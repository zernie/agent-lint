# agent-lint

Validate your CLAUDE.md, audit your feedback loops, and generate lint rules from PR comments.

Companion repo for [Feedback Loop Is All You Need](https://zernie.com/blog/feedback-loop-is-all-you-need).

## Why

AI coding agents (Claude Code, Copilot, Cursor) work best when they get deterministic feedback — linters, type checkers, CI, observability. Without it, they hallucinate patterns, drift from conventions, and produce code that "works" but doesn't fit your codebase.

`agent-lint` helps you close the feedback loop:

1. **Validate your CLAUDE.md** — ensure every rule is either enforced by a linter or explicitly marked as guidance-only
2. **Audit your repo's maturity** — score how well your feedback loops support AI-assisted development
3. **Generate lint rules from PR comments** — turn recurring review feedback into automated enforcement

## Install

```bash
npm install -D agent-lint
```

## CLAUDE.md Validator

The main attraction. Parses your CLAUDE.md and checks that every rule section (### heading) has either an `**Enforced by:**` annotation or a `**Guidance only**` marker.

```bash
# Run from your project root
npx agent-lint validate

# Or specify a path
npx agent-lint validate --claude-md ./docs/CLAUDE.md
```

### Expected CLAUDE.md format

```markdown
### Always use barrel file imports
**Enforced by:** `eslint/no-restricted-imports`
**Why:** Prevents import path drift during refactoring.

### Use Tailwind spacing scale, no magic numbers
**Guidance only** — cannot be mechanically enforced
**Why:** Ensures visual consistency across the design system.
```

Rules missing both annotations cause a non-zero exit code, making this CI-friendly.

### Programmatic API

```typescript
import { validate } from "agent-lint";

const result = validate({ projectRoot: "./my-project" });
console.log(result);
// {
//   rules: [...],
//   enforced: 3,
//   guidanceOnly: 1,
//   missing: 0,
//   total: 4,
//   valid: true
// }
```

## Claude Code Skills

Copy the skills to your `.claude/skills/` directory to use them:

```bash
mkdir -p .claude/skills
cp node_modules/agent-lint/skills/*.md .claude/skills/
```

### `/audit-feedback-loop`

Scans your repo and scores its feedback loop maturity:

| Level | Name | Description |
|-------|------|-------------|
| 0 | Vibes | No CI, no linters, no CLAUDE.md |
| 1 | Guardrails | CI + standard linters, no custom rules |
| 2 | Architecture as Code | Custom lint rules + enforced CLAUDE.md |
| 3 | The Organism | CI + custom rules + visual tests + observability + scheduled agents |

Works with any language — detects ESLint, Ruff, Clippy, golangci-lint, RuboCop, and more.

### `/pr-to-lint-rule`

Takes a natural language description of a recurring PR comment and generates:

- A lint rule for your language/toolchain (ESLint, Ruff, Clippy, go/analysis, etc.)
- Test cases
- Integration instructions
- A CLAUDE.md annotation block

Example: `/pr-to-lint-rule we keep telling people not to import directly from antd, use our design system barrel file instead`

## ESLint Plugin

Ships an empty ESLint plugin shell (`eslint-plugin-agent-lint`) for rules generated by `/pr-to-lint-rule`:

```javascript
// eslint.config.js
import agentLint from "agent-lint/plugin";

export default [
  {
    plugins: {
      "agent-lint": agentLint,
    },
    rules: {
      // Rules generated by /pr-to-lint-rule go here
    },
  },
];
```

## GitHub Actions

Copy the workflow template to your repo:

```bash
mkdir -p .github/workflows
cp node_modules/agent-lint/.github/workflows/feedback-loop.yml .github/workflows/
```

It runs: lint → type check → tests → CLAUDE.md validation.

## Maturity Levels

From the article — the four levels of feedback loop maturity:

### Level 0: Vibes
The agent writes code, you eyeball it. No automated checks. Scales to about one file.

### Level 1: Guardrails
Standard CI: ESLint defaults, TypeScript strict mode, basic tests. The agent gets red/green signals but can't learn your conventions.

### Level 2: Architecture as Code
Custom lint rules encode your team's decisions. CLAUDE.md rules reference actual enforcement. The agent understands *why* things are done a certain way.

### Level 3: The Organism
Everything is instrumented. Visual regression tests catch UI drift. Observability SDKs flag runtime issues. Scheduled agents monitor and maintain. The codebase evolves with feedback at every layer.
